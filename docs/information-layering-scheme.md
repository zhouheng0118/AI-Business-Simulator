# 信息分层方案

基于 Harvard PON 角色扮演模拟教学设计框架 + Bloom's 认知分类法。

---

## 一、核心模型

```
基本层（Basic Layer）
= Harvard PON "General Information"
= Bloom's 记忆 / 理解层
= 学生开场前就能看到，用于理解"问题是什么、谁负责什么"

隐藏层（Hidden Layer）
= Harvard PON "Confidential Instructions"
= Bloom's 分析 / 评估层
= 学生通过与 agent 对话挖掘，用于发现"问题比表面更复杂"
```

---

## 二、完整四步处理流程

```
教授上传 Case
      │
      ▼
┌─────────────────────────┐
│  步骤一：相关性筛选       │  ← 以教学目标为过滤器
│  只保留服务于教学目标     │
│  的信息，丢弃其余内容     │
└─────────────────────────┘
      │
      ▼
┌─────────────────────────┐
│  步骤二：决策树 A         │  ← 判断每条信息属于哪一层
│  基本层 vs 隐藏层         │
└─────────────────────────┘
      │
      ▼
┌─────────────────────────┐
│  步骤三：决策树 B         │  ← 仅针对隐藏层信息
│  隐藏层解锁难度分级        │  ← 判定 Level 1 / 2 / 3
└─────────────────────────┘
      │
      ▼
┌─────────────────────────┐
│  步骤四：教授审阅与编辑    │  ← 可视化界面，人工校正
│  信息可视化 + 编辑功能     │
└─────────────────────────┘
      │
      ▼
   Playbook 确认发布
```

## 三、步骤一：相关性筛选

**触发条件：** 教授在上传 Case 时填写的教学目标（Teaching Goals）和教学问题（Discussion Question）。

**筛选标准（对每一条 case 信息问这个问题）：**

> 这条信息是否影响学生回答教授设定的教学问题的能力？

```
[一条 Case 信息]
      │
      ▼
这条信息与教学目标直接相关，
或影响学生回答教学问题吗？
      │
      ├── 是 ──→ 保留，进入步骤二（决策树 A）
      │
      └── 否 ──→ 丢弃，不进入分层流程
```

**示例：**

| 教学目标 | 保留的信息 | 丢弃的信息 |
|----------|-----------|-----------|
| 评估市场进入时机决策 | 竞争对手动态、当地牌照周期、现金储备 | 公司 Logo 设计历史、创始人个人经历 |
| 分析单位经济模型可行性 | 单位成本、定价策略、客户获取成本 | 公司办公室地址、团队文化活动 |

---

## 四、步骤二：决策树 A（基本层 vs 隐藏层）

对筛选后的每一条信息，依次回答以下四个问题：

```
[一条信息]
    │
    ▼
┌─────────────────────────────────────────────────────┐
│ 公开测试                                             │
│ 这是公开可查、或所有参与者逻辑上都应知道的信息吗？      │
└─────────────────────────────────────────────────────┘
    │ 是 ──────────────────────────────────→ 【基本层】
    │ 否
    ▼
┌─────────────────────────────────────────────────────┐
│ 前提测试                                             │
│ 学生不知道这个，连"该向谁问什么"都想不到吗？           │
└─────────────────────────────────────────────────────┘
    │ 是 ──────────────────────────────────→ 【基本层】
    │ 否
    ▼
┌─────────────────────────────────────────────────────┐
│ 转折测试                                             │
│ 这个信息一旦揭露，会改变学生的分析结论或建议方向吗？    │
└─────────────────────────────────────────────────────┘
    │ 否 ──────────────────────────────────→ 【省略】（低价值，不收录）
    │ 是
    ▼
┌─────────────────────────────────────────────────────┐
│ 现实测试                                             │
│ 这个利益相关者在现实中有合理动机不主动说出这件事吗？    │
└─────────────────────────────────────────────────────┘
    │ 否 ──────────────────────────────────→ 【基本层】
    │ 是
    ▼
                                              【隐藏层】
                                         → 进入步骤三（决策树 B）
```

**四个测试的判断逻辑：**

| 测试 | 判断方向 | 核心问题 |
|------|----------|----------|
| 公开测试 | 信息来源 | 报纸 / 官网能查到吗？ |
| 前提测试 | 认知前提 | 没有这个，学生连起点都找不到吗？ |
| 转折测试 | 影响力度 | 知道之后，结论会变吗？ |
| 现实测试 | 人物动机 | 这个人有理由藏着这件事吗？ |

## 五、步骤三：决策树 B（隐藏层解锁难度分级）

进入隐藏层的信息，还需要确定被挖掘出来的难度等级：

```
[一条隐藏信息]
    │
    ▼
┌─────────────────────────────────────────────────────┐
│ 认知跨度测试                                         │
│ 学生只需问到正确话题，无需追问或前置条件，就能触发吗？  │
└─────────────────────────────────────────────────────┘
    │ 是 ──────────────────────────────────→ 【Level 1】
    │ 否
    ▼
┌─────────────────────────────────────────────────────┐
│ 假设质疑测试                                         │
│ 学生需要质疑基本层中某个数字 / 假设的合理性才能触发吗？ │
│ 或者需要先从另一个 agent 获得信息才能问出来吗？        │
└─────────────────────────────────────────────────────┘
    │ 是 ──────────────────────────────────→ 【Level 2】
    │ 否
    ▼
┌─────────────────────────────────────────────────────┐
│ 跨角色依赖测试                                       │
│ 学生需要把两个以上 agent 的说法拼在一起，             │
│ 才会意识到要提出这个问题吗？                          │
└─────────────────────────────────────────────────────┘
    │ 是 ──────────────────────────────────→ 【Level 3】
    │ 否
    ▼
                                              【Level 2】（默认）
```

### 编辑功能说明

| 功能 | 操作方式 | 说明 |
|------|----------|------|
| 编辑信息内容 | 点击卡片 [编辑] 按钮 | 可修改信息文本 |
| 编辑解锁条件 | 编辑隐藏层卡片时同步显示 | 仅隐藏层卡片有此字段 |
| 修改解锁难度 | 编辑隐藏层卡片时下拉选择 | L1 / L2 / L3 切换 |
| 移动信息层级 | 卡片编辑界面内 [移至基本层] / [移至隐藏层] | 跨层移动 |
| 添加新信息 | 点击各区域底部 [+ 添加] 按钮 | 手动补充 agent 未生成的信息 |
| 删除信息 | 编辑状态下 [删除] 按钮 | 移除不需要的条目 |

## 七、各层信息内容参考

### 基本层：五类信息

每个 agent 持有 4–5 条，覆盖以下类型：

| 类型 | 内容说明 | 示例 |
|------|----------|------|
| 公司 / 市场背景 | 公开的公司规模、业务模式、市场环境 | "公司成立于 2019 年，主营 B2B SaaS，现有企业客户 300 家" |
| 核心决策情境 | 当前正在考虑的决策及其紧迫性 | "正评估是否在 2026 年 Q2 前进入东南亚市场" |
| 各 agent 职责声明 | 该利益相关者的官方职责范围 | "CFO 负责融资策略与现金流管理" |
| 表面可见的张力 | 已知的矛盾或压力，但未揭示根因 | "增长目标与当前运营资源存在明显缺口" |
| 公开的关键数字 | 官网、财报或公开报道中可查的数据 | "目前月活用户 50 万，同比增长 40%" |

**基本层质量检查：**
- 每条信息是否帮助学生理解"问题是什么"？
- 有没有不小心把隐藏层内容混入？（用决策树 A 复验）
- 各 agent 的基本层信息是否彼此一致、无矛盾？

---

### 隐藏层：三级信息

全局 8–12 条，跨 agent 分布。

**Level 1｜认知跨度低（3–4 条）**

学生问到正确话题即可触发，鼓励探索行为。

| 信息类型 | 现实隐藏动机 | 示例 |
|----------|------------|------|
| 内部财务压力 | 避免在早期谈判中暴露弱势 | "实际现金储备只够运营 4 个月" |
| 已知但未公开的竞争动态 | 避免引发团队恐慌 | "主要竞争对手已在目标城市签约 3 家核心供应商" |
| 利益相关者的角色边界 | 不想主动限制自己的话语权 | "COO 对技术路线有最终否决权，但 CEO 通常不提" |

解锁条件示例：`学生询问当前现金储备或融资计划`

---

**Level 2｜需要质疑假设或前置信息（4–5 条）**

学生需要发现基本层里有"站不住脚"的地方，或先从另一个 agent 处获得线索。

| 信息类型 | 现实隐藏动机 | 示例 |
|----------|------------|------|
| 乐观预测背后的脆弱假设 | 没人想打破共识 | "30% 增长预测基于 2 个尚未签约的大客户" |
| 内部执行层与战略层的分歧 | 维护对外的一致形象 | "COO 认为 Q2 上线时间表不可行，但未对 CEO 正式反对" |
| 法规或合规的真实约束 | 本地专家觉得这不是自己主动要说的 | "当地牌照实际审批周期 18 个月，官网显示 6 个月" |
| 客户留存的真实数据 | 避免动摇投资人信心 | "头部客户续约率仅 60%，而非对外宣传的 85%" |

解锁条件示例：`学生质疑增长预测的基础假设，或追问该预测基于哪些已确认的客户`

---

**Level 3｜需要跨角色交叉验证（1–3 条）**

学生需要将两个以上 agent 的说法拼合，才能意识到要提出这个问题。

| 信息类型 | 现实隐藏动机 | 示例 |
|----------|------------|------|
| 跨部门的战略矛盾 | 双方都在等对方先开口 | "CEO 承诺董事会 Q2 上线，但 CFO 知道当前预算无法支撑这一时间表" |
| 利益相关者的个人利益冲突 | 主动说等于暴露自己的动机 | "CMO 的 bonus 与本季度 GMV 直接挂钩，推动的激进增长目标有个人利益因素" |
| 隐藏的外部依赖 | 各方都认为是别人该说的 | "整个东南亚扩张计划依赖一个未签约的本地合作伙伴，但 CEO 和 COO 都未主动提及" |

解锁条件示例：`学生已从 CEO 处获知 Q2 上线承诺，再向 CFO 追问当前预算是否支撑该时间表`

## 八、完整分层结构示意

```
一个 Case 处理完成后的信息分布：

基本层（每个 agent 4–5 条，共约 20–25 条）
├── 公司 / 市场背景        × 2–3 条（所有 agent 共享）
├── 核心决策情境           × 1–2 条（所有 agent 共享）
└── 角色专属职责与事实     × 2 条（每个 agent 不同）

隐藏层（全局 8–12 条，跨 agent 分布）
├── Level 1  简单解锁      × 3–4 条
├── Level 2  中等解锁      × 4–5 条
└── Level 3  跨角色解锁    × 1–3 条
```

## 九、校验原则

> 决策树判，内容表验，冲突时决策树赢。
